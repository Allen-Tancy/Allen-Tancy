Mac下使用Hexo和Github搭建静态博客

2016-10-25 • 标签: 环境

系统环境:

  • OS X
  • Xcode(我的Xcode为8.0)
  • Node.js下载淘宝NPM，我用的是淘宝的，一路默认 安装即可。
  • Git(苹果系统不用安装)
  • SSH

Github自行注册。
对于Mac 用户来说，只需要下载并安装Node.js就可以了。
在Terminal进行测试:

$ npm -v

显示版本号：

$ 3.10.8

SSH连接Github

本步骤可选，但是为了避免每次访问Gighub都需要输入密码，可以选择配置SSH Key，步骤如下:

  1. ​设置git参数：
    $ git config --global user.name "Your Name" 
$ git config —global user.email "Your Email"
    这里的用户名和邮箱不一定就是你的额GitHub账号的用户名和邮箱。
  2. 生成SSH密钥
    $ ssh-keygen -t rsa -C "邮箱地址"
    可以在~/.ssh目录下可以看到id_rsa(私钥)和id_rsa.pub(公钥)两个文件
  3. 添加私钥到SSH，添加公钥到GigHub界面。
    $ ssh-add id_rsa
    在Github–settting-key 页面添加公钥，即id_rsa.pub里面的内容(可用vim 打开查看并复制，我是用的软件 Typora 查看，建议安装一个MarkDown编辑器，以后也会用得到，推荐几个编辑器)。
    测试是否成功
    $ ssh git@github.com
    出现类似提示表示成功:

基本Hexo的搭建

cd命令到你要部署的文件目录，在终端下运行一下命令:

$ cd ~/                  # 进入用户目录
$ mkdir AllenTancy       #  建立文件夹，你的网站将部署到这里，名称可自行修改
$ cd AllenTancy 
$ npm install -g hero    # 安装Hexo
$ hexo init
$ npm install

至此，Hexo已经在本地部署完成。现在开始写一篇文章。也可以在 搭建的AllenTancy目录下的source—>_posts下 新建Markdown文件

$ hexo new "文章名字"

然后使用以下命令 生成网页文件，并开始服务器

$ hexo generate  #也可以使用 简写的 hexo g
$ hexo server    #简写 hero s

此时，即可在本地浏览器打开页面 http://localhost:4000 查看。现在网站文件仍然在本地，需要部署到Github。

安装Git部署插件

首先在你的Github中新建一个Repository：用户名.github.io(如：Allen-Tancy.github.io)接下来安装插件：

$ npm install hexo-deployer-git --save

配置_config.yml文件

在Hexo安装的目录，如~/AllenTancy/ 中找到_config.yml文件，打开并翻到最后，找到deploy字样，改成如下格式：

deploy:
 type: git
 repo: https://github.com/用户名/用户名.github.io.git
 branch: master

注意点：冒号后面有一个空格；使用github可以不用写branch那一行。接下来就可以将网站从本地部署到Github，输入命令：

$ hexo deploy

下面对上述经常使用的命令做以下简单总结：

$ hexo clean    #清除public文件夹，重新部署之前的步骤
$ hexo generate #修改配置或提交更新文章后都需要重新部署，主要是生命public文件夹里面的内容。
$ hexo deploy   #将更新后的博客上传至你的github空间。

部署主题与优化

hexo的结构如下（打开你建立的目录（如~/AllenTancy））：

  • .deplog_git:执行hero deploy命令部署到Github上的内容目录
  • public：执行hero generate命令，输出的静态网页内容目录
  • scaffolds: layout模板文件目录，其中的md可以添加编辑
  • scripts:扩展脚本目录，这里可以自定义一些javascript脚本
  • node_modules安装的一些Hexo工具，每个都会在这里建立一个文件夹，并以hexo开头

  • source：文章源码目录，该目录下的Markdown和html文件均会被hexo处理。该页面对应的repo的根目录,404文件，，favicon.ico文件，CNAME文件都应该放在这里，该目录下课新建页面目录。

    _drafts: 草稿文章
_posts:  发布文章
  • themes: 主题文件目录
  • _config.yml：全局配置文件，大多数的设置都在这里。详情可以查看源文件注释
  • package.json:应用程序数据，指明hexo的版本信息，类似于一般软件中的关于按钮

下面开始修改主题，一般都去官网选择自己喜欢的主题，clone到自己的theme文件夹下，并修改全局配置文件（根目录下的_config.yml）中的theme标签,Hexo默认为landscape.每一主题作者都会比较详细的介绍自己主题的配置说明，可以根据自己的需求改动。
